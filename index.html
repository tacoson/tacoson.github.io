10分鐘週記-綜合所得稅

2022-07-12

給自己一個挑戰，10分鐘來完成紀錄一週大小事，這週把今年報稅的大小事整理一下，每年到五月的時候同樣的事情都會困擾一次，究竟我能否找到更好的報稅方式。
以今年度(2022)的小弟我報稅方式，單身、無扶養人口、標準扣除額(120,000)，我想應該是相同收入下最悲壯的繳稅方式了，尤其我還是北漂族，所得級距進入12%的深淵。
目前想到的方法如下:

增加扶養人口

參考網址: 2022 所得稅扶養親屬扣除額多少？節稅優缺點全盤分析
能夠扣除的細節就不贅述，原則上扶養人口越多，若被扶養者增加的免稅額(70歲以下8,8000；70歲以上132,000)+扣除額(儲蓄扣除額270,000…等)+基本生活費差額(192,000 * 人數再扣掉前面的差額)大於因扶養產生的所得，就會有節稅的效果，這也是一般人的作法。
對我來說卻是親子溝通問題，可行性不高，父母的收入不清楚實際所得數字(但我的數字父母知道)，變成單方面接受父母規劃的結果，但理論上公務人員退休金免稅額有78.1萬，超過的部分才會進行課稅，我不太認為我家狀況是會多出多少(除非退休還能領破百萬)，但難就難在傳統家庭這部分就有點難開口溝通了，子女主動探索隱私是不是逾越了自己的權限!!
我個人覺得是同理心問題，將來自己若有下一代，一旦子女開始工作，我一定好好把職場中，稅務這一方面的概念及家裡實際情況，分析給他聽，站在彼此的立場選擇對節稅最大公約數出發，互相理解而非單向輸出。
所謂扶老攜幼，老的不行，趕進度來有後代也不失為壞方法，當然也僅紙上談兵，也得生得出來才能成真。

標準扣除額改列舉扣除額

這部分就較無可置喙，畢竟標準扣除額是12萬，我的人身保險應可報滿24000，健保費大概18000，最大的癥結點在於房租，這又陷入另外一個難處，我租的不夠貴，目前住的是月租金5500的小套房，縱使申報列舉，可能也是接近12萬左右，差異並不會太明顯。

勞退自提

參考網址: 111年勞保、健保、勞退保費對照表
日前想到有效的方法，便是增加勞退自提的比率(每年1月和7月可以調整自提比率)，法規目前規範個人可以最高選擇6%進行提撥，這部分免納入所得計算，以目前我的級距，每個月提撥5526，明年節稅接近7957 (5526 * 12 * 0.12)。

這有點強迫儲蓄和節稅的雙重效果。

HTTP 基礎知識

裡面有提到.js的套件(‘request’和’http’)可以玩玩看( 但經查看 request 已經廢棄使用)

chrome 開發人員工具
  • CTRL + SHIFT + I -> [Network] 可以觀察 request 和 reponse 的內容
    HTTP Method
  • GET 跟SERVER要一個連結或者圖片，通常只是要獲取資訊，因此沒有request BODY。
  • POST 需要執行一些動作，把資料放在request body
  • PUT 取代整個 request (POST和PUT差異在於POST動作為create、PUT動作為replace)
  • PATCH 修改部分 request (replace某些欄位)
  • DELETE 刪除資源
  • HEAD 只獲取 request的 head
  • OPTION 可以了解Sever提供那些溝通方法

Idempotent Method: GET PUT DELETE HEAD (此特性決定是否能夠Retry)
Safe method : GET HEAD (此特性決定是否能夠Cache)

傳遞資料的方式有兩種:GET和POST，通常一般表單可以用GET直接透過URL參數傳遞，假設不希望出現在網址上，用POST處理，例如使用者登入帳密。
當然因為POST還是明碼傳輸，比較安全作法為加上SSL憑證，使用HTTPS傳輸

URI語法:
[協定名]://[使用者名稱]:[密碼]@[主機名]:[埠]/[路徑]?[查詢參數]#[片段ID]

Response Status code
  • 1XX: 稍等
  • 2XX: 成功
  • 3XX: 重新導向
  • 4XX: 客戶端錯誤
  • 5XX: 伺服器錯誤

開放網路軟體安全計畫
參考網址: 開放網路軟體安全計畫
參考網址: GSS公司名詞解釋
簡稱OWASP （Open Web Application Security Project）是一個開放社群、非營利性組織，全球目前有82個分會，其主要目標是研議協助解決網路軟體安全之標準、工具與技術文件，長期致力於協助政府或企業瞭解並改善應用程式的安全性。美國聯邦貿易委員會（FTC）更強烈建議所有企業務必遵循OWASP所發佈的十大網路弱點防護守則，美國國防部亦將此守則列為最佳實務，就連國際信用卡資料安全技術PCI標準更將其列為必要元件。

法規筆記

風險評鑑與管理

  • 鑑別系統維運單位及資安風險等級(系統等級(機密性、可用性、完整性) x可能性 x衝擊性)，並評估可容忍中斷時間。

資訊安全政策

  • 制定公司政策、導入資訊安全管理系統。

安全組織

  • 指定副總或高層主管人員進行推動，並規範從業人員資安專業訓練課程時數及維持相當專業證照。

資產分類與控制

  • 資訊資產清冊，並定期檢視妥適性。

人員安全

  • 保密切結、設定內部稽核人員、定期參加講習並留存紀錄。

實體與環境安全

  • 刷卡、CCTV、機房防火防震、不斷電系統、發電機、報廢程序(機密性、敏感性資料及授權軟體予以刪除)並留存紀錄。

通訊與作業管理

  • 定期評估作業系統、網站伺服器、瀏覽器、防火牆、防毒版本並留存文件。
  • 公司網路依用途區隔DMZ、營運環境、測試環境不同網段進行管理。
  • 系統應僅開啟必要之服務及程式，未使用之服務功能應關閉。
  • 遠端連線管理進公司進行內部作業必須進行管控，留存相關維護紀錄供定期稽核。
  • 公司應防止未經授權設備使用內部網路。
  • 應建立防火牆，防火牆進出紀錄及備份至少三年。
  • 定期檢視並維護防火牆存取管控設定，並留存相關檢視紀錄。
  • 應用程式採加密方式(例如SSL)、採取多因子驗證(下單憑證、綁定裝置、OTP、生物辨識等機制)以確保為本人。
  • 全面使用憑證機制、訂定憑證交付程序避免非本人取得憑證。
  • 應安裝防毒軟體、並定期更新。應定期對電腦系統及資料儲存媒體進行病毒掃描，防毒要包含個人端(攜帶型及營業處所之電腦) 。
  • 勿開啟來歷不明之電子郵件，尤其是有帶有執行檔之附件。
  • 應建立公司電子郵件使用規範及建立郵件過濾機制。
  • 應建立上網管制措施，以避免下載惡意程式。
  • 應偵測釣魚網站及惡意網站連結並提醒客戶防範網路釣魚。
  • 可定期辦理社交工程演練，對誤開啟信件或連結之人員進行教育訓練，並留存相關紀錄。
  • 應依照資訊安全分級定期對核心系統辦理滲透測試、資安健檢、入侵偵測(事件收集、異常分析、偵測並判斷攻擊行為)及防禦機制、設置應用程式防火牆。
  • 電腦設備管理應保存維護紀錄。
  • 重要系統使用多因子機制及安全組態基準(如密碼長度、更新期限) 。
  • 備份機制及回存測試機制、電腦操作日誌應主管檢核，操作人員不可與主管同一人。
  • 系統容量每年定期由內部或外部專業機構評估。

存取控制

  • 委外作業人員使用內部網路資源，應有安全管制措施。
  • 定期審查並檢討久未使用之使用者權限。
  • 密碼以亂碼方式儲存、密碼錯誤次數達三次予以中斷連線、公司應使用優質密碼設定(長度6字元以上，且具有文數字及符號) 。
  • 電腦系統稽核紀錄資訊應包含使用者識別碼、登入之日期時間，並保留三年。
  • 依照個資保護法妥善保護客戶及內部人個人資料。

系統開發及維護

  • 應用系統規劃時應將資訊安全需求納入分析及規格。
  • 檢查輸入資料。
  • 使用合法授權軟體。
  • 各項文件及手冊應經是當維護與控制。
  • 應用程式異動管理:正式作業跟測試作業程式、資料、作業控制指令應分開存放。
  • 定期辦理弱掃找出潛在系統弱點。
  • 程式原始碼安全規範:避免惡意軟體、確保完整性、函式庫維護、輸入字串安全性檢查避免注入攻擊。
  • 行動應用程式安全檢測，可委由財團法人全國認證基金會(TAF)認證合格之第三方檢測實驗室完成功過資安檢測。檢測後若有更新之必要，則以OWASP MOBILE TOP 10之標準為依據進檢測。

持續營運管理

  • 明確訂定(例如電腦設備、通訊設備、電力系統、資料庫、電腦作業系統等備援及回復計畫)故障復原程序。
  • 故障復原程序應周期性測試，測試後召開檢討會議。
  • 公司應擬定營運持續計畫(含啟動條件、參與人員、緊急程序、備援程序、維護時間表、教育訓練、職責說明、往來單位之應變規劃)、資安訊息通報之機制、訂定分散式阻斷服務攻擊(DDoS)防禦與應變作業程序。

符合性

  • 應辦理資訊安全查核作業(內部辦理或委外)，並留存查核紀錄。公司是否針對前開之資訊安全查核報告辦理追蹤改善情形。

新興科技應用

  • 雲端服務使用者，應訂定雲端運算服務運作安全規範，如不符需求之處，須有補償性措施。
  • 雲端服務提供者，應訂定雲端運算服務安全管控措施(法律遵循、權限控管、權責歸屬及資訊安全防護)，機敏資料要加密(HTTPS or SFTP)。
  • 評估開放員工使用社群媒體並估風險程度(資料外洩、社交工程、惡意程式攻擊)。
  • 公司應訂定自攜行動裝置的安全規範及管理辦法。

其他

  • 資料提供或公告相關規範。

主機共置服務管理

  • 進出主機共治機房應進行申請並配合清點及留存紀錄。
