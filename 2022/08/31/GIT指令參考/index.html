GIT 已經紅滿久的，10年前對他的概念只知道是分散式的版控軟體，但對於實際上的分支系統設計並不清楚，官網的說明文件寫得滿有知識含量，值得一看
目前我只有看第2章Git基礎及第3章使用GIT分支，先把看到的指令做個剪貼本:

  • 檔案放置在有Working Directory(WD), Staging Area(預存區), .git repository(Repository)，三個區域

    檔案狀態
    Modified 檔案已經變更但是尚未存於本地端資料庫
    Staged 已預存這檔案會被存在下次快照
    Commited 已儲存至本地端(local)資料庫

  • 預存區實體是單一檔案

  • 基本 Git流程，在工作目錄修改檔案；預存檔案並將快照新增至預存區；做提交動作，這會讓預存區的檔案快照永久存在Git目錄中

  • Git的分支模型和其他 (Version Control System, 以下簡稱VCS) 與眾不同的地方，便是每次新增的快照都會以”增加”存在，而並非以DIFF方式，大幅度降低本地資料庫故障機率。

  • 單次提交的目錄結構(Tree)，也可視為快照，有使用SHA當雜湊碼物件識別

  • 提交和親代提交的關係(注意master和HEAD指標意義)

  • Master是第一個建立的分支；HEAD並非像其他CVS下git branch指令會切換”目前”分支。
