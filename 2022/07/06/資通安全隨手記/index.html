資通安全隨手記

2022-07-06
字數統計: 2.3k   |   閱讀時長≈ 7 分鐘

本周因為研讀資通安全檢查機制，做個筆記備忘。
資訊安全政策，實務上將企業資本額大小來區分不同級別分類，並將應辦理事項逐項描述以提供企業進行辦理，也有利於稽核方進行稽核。
一定資本規模以上的企業必須導入資訊安全系統，後續則依照檢查機制來訂定一致性的要求。

資安名詞

資安健檢
參考網址: 資安健檢
網路架構檢視、網路惡意活動檢視(有線)、使用者端電腦惡意活動檢視、伺服器主機惡意活動檢視、目錄伺服器設定檢視、防火牆連線設定檢視、政府組態基準(GCB)檢視及資料庫安全檢視等八大項資安專業檢測服務，涵蓋網路面、端點面、設定面及資料面等四大面向。

滲透測試
參考網址: 滲透測試
滲透測試服務是模擬真實駭客，扮演攻擊方 (Red Team) 挑戰系統防護強度，找出可入侵的漏洞與途徑，用以協助企業檢測網路是否存在安全上的隱憂，本服務係利用系統化工具與流程，結合多年實戰經驗之資訊安全專家、資訊安全技術及資安工具，對目標系統進行深入的安全強度測試，並利用各種滲透技術、攻擊手法、弱點攻擊程式、網站漏洞等技巧來對受測主機進行遠端滲透測試，並於檢測完成後提供專業評估報告、完整地記錄整個測試過程與細節、改善建議及專業諮詢，協助企業強化資安防禦措施。

紅隊演練
參考網址: 紅隊演練
A.K.A. 不特定範圍、形式的滲透測試。

行動應用資安聯盟
參考網址: 行動應用資安聯盟
該組織公布之行動應用程式基本資安檢測基準項目。

開放網路軟體安全計畫
參考網址: 開放網路軟體安全計畫
參考網址: GSS公司名詞解釋
簡稱OWASP （Open Web Application Security Project）是一個開放社群、非營利性組織，全球目前有82個分會，其主要目標是研議協助解決網路軟體安全之標準、工具與技術文件，長期致力於協助政府或企業瞭解並改善應用程式的安全性。美國聯邦貿易委員會（FTC）更強烈建議所有企業務必遵循OWASP所發佈的十大網路弱點防護守則，美國國防部亦將此守則列為最佳實務，就連國際信用卡資料安全技術PCI標準更將其列為必要元件。

法規筆記

風險評鑑與管理

  • 鑑別系統維運單位及資安風險等級(系統等級(機密性、可用性、完整性) x可能性 x衝擊性)，並評估可容忍中斷時間。

資訊安全政策

  • 制定公司政策、導入資訊安全管理系統。

安全組織

  • 指定副總或高層主管人員進行推動，並規範從業人員資安專業訓練課程時數及維持相當專業證照。

資產分類與控制

  • 資訊資產清冊，並定期檢視妥適性。

人員安全

  • 保密切結、設定內部稽核人員、定期參加講習並留存紀錄。

實體與環境安全

  • 刷卡、CCTV、機房防火防震、不斷電系統、發電機、報廢程序(機密性、敏感性資料及授權軟體予以刪除)並留存紀錄。

通訊與作業管理

  • 定期評估作業系統、網站伺服器、瀏覽器、防火牆、防毒版本並留存文件。
  • 公司網路依用途區隔DMZ、營運環境、測試環境不同網段進行管理。
  • 系統應僅開啟必要之服務及程式，未使用之服務功能應關閉。
  • 遠端連線管理進公司進行內部作業必須進行管控，留存相關維護紀錄供定期稽核。
  • 公司應防止未經授權設備使用內部網路。
  • 應建立防火牆，防火牆進出紀錄及備份至少三年。
  • 定期檢視並維護防火牆存取管控設定，並留存相關檢視紀錄。
  • 應用程式採加密方式(例如SSL)、採取多因子驗證(下單憑證、綁定裝置、OTP、生物辨識等機制)以確保為本人。
  • 全面使用憑證機制、訂定憑證交付程序避免非本人取得憑證。
  • 應安裝防毒軟體、並定期更新。應定期對電腦系統及資料儲存媒體進行病毒掃描，防毒要包含個人端(攜帶型及營業處所之電腦) 。
  • 勿開啟來歷不明之電子郵件，尤其是有帶有執行檔之附件。
  • 應建立公司電子郵件使用規範及建立郵件過濾機制。
  • 應建立上網管制措施，以避免下載惡意程式。
  • 應偵測釣魚網站及惡意網站連結並提醒客戶防範網路釣魚。
  • 可定期辦理社交工程演練，對誤開啟信件或連結之人員進行教育訓練，並留存相關紀錄。
  • 應依照資訊安全分級定期對核心系統辦理滲透測試、資安健檢、入侵偵測(事件收集、異常分析、偵測並判斷攻擊行為)及防禦機制、設置應用程式防火牆。
  • 電腦設備管理應保存維護紀錄。
  • 重要系統使用多因子機制及安全組態基準(如密碼長度、更新期限) 。
  • 備份機制及回存測試機制、電腦操作日誌應主管檢核，操作人員不可與主管同一人。
  • 系統容量每年定期由內部或外部專業機構評估。

存取控制

  • 委外作業人員使用內部網路資源，應有安全管制措施。
  • 定期審查並檢討久未使用之使用者權限。
  • 密碼以亂碼方式儲存、密碼錯誤次數達三次予以中斷連線、公司應使用優質密碼設定(長度6字元以上，且具有文數字及符號) 。
  • 電腦系統稽核紀錄資訊應包含使用者識別碼、登入之日期時間，並保留三年。
  • 依照個資保護法妥善保護客戶及內部人個人資料。

系統開發及維護

  • 應用系統規劃時應將資訊安全需求納入分析及規格。
  • 檢查輸入資料。
  • 使用合法授權軟體。
  • 各項文件及手冊應經是當維護與控制。
  • 應用程式異動管理:正式作業跟測試作業程式、資料、作業控制指令應分開存放。
  • 定期辦理弱掃找出潛在系統弱點。
  • 程式原始碼安全規範:避免惡意軟體、確保完整性、函式庫維護、輸入字串安全性檢查避免注入攻擊。
  • 行動應用程式安全檢測，可委由財團法人全國認證基金會(TAF)認證合格之第三方檢測實驗室完成功過資安檢測。檢測後若有更新之必要，則以OWASP MOBILE TOP 10之標準為依據進檢測。

持續營運管理

  • 明確訂定(例如電腦設備、通訊設備、電力系統、資料庫、電腦作業系統等備援及回復計畫)故障復原程序。
  • 故障復原程序應周期性測試，測試後召開檢討會議。
  • 公司應擬定營運持續計畫(含啟動條件、參與人員、緊急程序、備援程序、維護時間表、教育訓練、職責說明、往來單位之應變規劃)、資安訊息通報之機制、訂定分散式阻斷服務攻擊(DDoS)防禦與應變作業程序。

符合性

  • 應辦理資訊安全查核作業(內部辦理或委外)，並留存查核紀錄。公司是否針對前開之資訊安全查核報告辦理追蹤改善情形。

新興科技應用

  • 雲端服務使用者，應訂定雲端運算服務運作安全規範，如不符需求之處，須有補償性措施。
  • 雲端服務提供者，應訂定雲端運算服務安全管控措施(法律遵循、權限控管、權責歸屬及資訊安全防護)，機敏資料要加密(HTTPS or SFTP)。
  • 評估開放員工使用社群媒體並估風險程度(資料外洩、社交工程、惡意程式攻擊)。
  • 公司應訂定自攜行動裝置的安全規範及管理辦法。

其他

  • 資料提供或公告相關規範。

主機共置服務管理

  • 進出主機共治機房應進行申請並配合清點及留存紀錄。
